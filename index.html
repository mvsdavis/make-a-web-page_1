Concepts

Lesson 1 Basics of The Web and HTML

HTML

  • HTML stands for Hypertext Markup Language.
  • HTML documents form the majority of the content on the web.
  • HTML documents contain text content which describes "what you see" and markup which describes "how it looks".

Tags and Elements

  • HTML documents are made of HTML elements.
  • When writing HTML, we tell browsers the type of each element by using HTML. Examples of tags are "b" for bold and "em" for italics.

Attributes

  • Attributes specify value. Examples are href for url and img for images.

Inline vs Block Elements

  • HTML elements are either inline or block.
  • Block elements form an "invisible box" around the content inside of them.
  • Inline tags include a, span, img,br and strong. An example of a block element is div.

Lesson 2 Creating a Structured Document HTML

The "tree-like structure" of HTML

  • The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them.
  • In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on.

The relationship between indented HTML and boxes

  • When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document.
  • The more indented an element is, the more deeply nested its corresponding box it will be.

Text Editors

  • When writing code, programmers use special text editors (scrathpadio, codepen and Sublime Text for example). Some text editors will automatically generate a closing HTML tag when you write an opening tag.

Lesson 3 Adding CSS Style and HTML Structure

CSS

  • CSS stands for Cascading Sytle Sheet
  • Most specific rule applied to every cascading element.
  • Selector defines elements which a particular syle will apply to. For example all paragraphs p{} or elements can be selected by class by using .description {attribute:value;color}. Selectors can be combined.

Reasons to Avoid Repitition

  • Avoiding Errors: When programmers don't avoid repetition and code the same thing over and over by using the copy and paste method to use certain styles/attributes may lead to errors.
  • Consistency: Assigning the same div to each lesson made for a uniformed look. It also makes changes easier to apply.

Box Model

  • Margin: transparent (space) between boxes
  • Border
  • Padding: clear area around content
  • Content

Things to remember

  • box-sizing:border-box will calculate border + padding + content but will not include margin
  • Add browser spcific prefix: webkit, MOZ or MS
  • Boxes can be calculated by pixels or percentages
  • Flexbox layout makes this easier: display.flex; (enter % less than 100)

Code, Test and Refine

  1. Look for natural boxes.
  2. Look for repeated styles and semantic elements
  3. Write HTML.
  4. Apply styles from biggest to smallest.
  5. Fix errors.
  6. Repeat #3, #4 and #5 as needed.
